# 背景介绍

## 社会背景

在大数据时代的背景下，未被使用的信息比例高达 99.4%，原因很大程度都是由于高价值的信息无法获取采集。因此，如何从大数据中采集出有用的信息已经是大数据发展的关键因素之一，数据采集可视为大数据产业的基石。

现代信息技术产业已经拥有 70 多年的历史，其发展的过程先后经历了几次浪潮。先是 20 世纪六七十年代的大型机浪潮，此时的计算机体型庞大，计算能力也不高。上世纪 80 年代以后，随着微电子技术和集成技术的不断发展，计算机各类芯片不断小型化，兴起了微型机浪潮，PC成为主流。上世纪末，随着互联网的兴起，网络技术快速发展，由此掀起了网络化浪潮，越来越多的人能够接触到网络和使用网络。

## 技术背景

本文主要阐述了离线大数据分析所使用的工具：Hadoop、MapReduce 分布式计算框架、Hive 数据仓库、Sqoop 数据迁移、echarts 可视化。

但是，我们将会着重介绍echarts 可视化部分，主要包括Spring，Spring MVC，MyBatis等新技术栈，前置的数据清洗部分将会弱化。

### Hadoop 简介

Hadoop 起源于 Apache Nutch 搜索引擎项目，旨在解决数十亿级网页的存储与索引难题。2003 年，谷歌先后发表GFS、MapReduce、BigTable 三篇论文，分别解决了海量数据的分布式存储、并行计算与实时查询问题，为 Hadoop生态系统的诞生奠定了理论基础。此后 Hadoop 逐渐发展为开源大数据处理的核心平台，本项目利用其 HDFS 存储与 MapReduce计算能力完成职位数据的清洗工作。

### Hadoop 的优势

高可靠性：因为 Hadoop 假设计算元素和存储会出现故障，因为它维护多个工作数据副本，在出现故障时可以对失败的节点重新分布处理。

高扩展性：在集群间分配任务数据，可方便的扩展数以千计的节点。

高效性：在 MapReduce 的思想下，Hadoop 是并行工作的，以加快任务处理速度。

高容错性：自动保存多份副本数据，并且能够自动将失败的任务重新分配。

### MapReduce 架构概述

MapReduce 是 Hadoop 生态的核心分布式计算模型，最初由 Google 设计，专门用于处理 TB 级以上的海量数据集。它借鉴了函数式编程思想，将复杂的分布式计算抽象为两个核心阶段：“Map（映射）”负责将原始数据分解并处理为中间键值对（Key/Value），“Reduce（归约）”则负责将相同键的值进行合并。该模型有效屏蔽了底层的分布式并行调度细节，使得开发者无需深厚的分布式编程基础，仅需定义具体的 Map 和 Reduce 函数即可实现大规模数据的并行计算，具有极高的工程实用价值。

图 1‑2 MapReduce执行流程

### Hive 数据仓库简介

Hive 是建立在 Hadoop 文件系统上的数据仓库，它提供了一系列工具，能够对存储在 HDFS 中的数据进行数据提取、转换和加载(ETL),这是一种可以存储、查询和分析存储在 Hadoop 中的大规模数据的工具。

Hive 定义了简单的类 SQL 查询语言，称为 HQL,它可以将结构化的数据文件映射为一张数据表，允许熟悉 SQL 的用户查询数据，也允许熟悉 MapReduce 的开发者开发自定义的 mapper 和 reducer 来处理复杂的分析工作，相对于 Java 代码编写的 MapReduce 来说，Hive 的优势更加明显。

Sqoop 是 Apache 旗下的一款开源工具，该项目开始于 2009 年，最早是作

## 数据预处理技术

现实世界中数据大体上都是不完整，不一致的脏数据，无法直接进行数据挖掘，或挖掘结果差强人意。为了提高数据挖掘的质量产生了数据预处理技术。数据预处理有多种方法：数据清理，数据集成，数据变换，数据归约等。这些数据处理技术在数据挖掘之前使用，大大提高了数据挖掘模式的质量，降低实际挖掘所需要的时间。

数据的预处理是指对所收集数据进行分类或分组前所做的审核、筛选、排序等必要的处理。

package position;

import org.codehaus.jettison.json.JSONArray;

import org.codehaus.jettison.json.JSONException;

import org.codehaus.jettison.json.JSONObject;

public class JobClean {

public static String cleanJob(JSONArray jobList) throws JSONException {

String jobCleanData = "";

for (int i = 0; i < jobList.length(); i++) {

JSONObject jsonListObject = jobList.getJSONObject(i);

String jobId = exField(jsonListObject, "encryptJobId");

String jobIdNew = parseString(jobId);

String jobName = exField(jsonListObject, "jobName");

String jobNameNew = parseString(jobName);

String salaryDesc = exField(jsonListObject, "salaryDesc");

String salaryNew = salaryResult(salaryDesc);

JSONArray skills = jsonListObject.optJSONArray("skills");

String skillNew = parseJSON(skills);

String jobDegree = exField(jsonListObject, "jobDegree");

String jobDegreeNew = parseString(jobDegree);

String cityName = exField(jsonListObject, "cityName");

String cityNameNew = parseString(cityName);

JSONArray welfareList = jsonListObject.optJSONArray("welfareList");

String welfareListNew = parseJSON(welfareList);

String jobExperience = jsonListObject.getString("jobExperience");

String jobExperienceNew = parseString(jobExperience);

if (i == jsonListObject.length()-1) {

jobCleanData = jobCleanData + jobIdNew + "\\001" + jobNameNew + "\\001" + salaryNew + "\\001" + jobExperienceNew + "\\001" + skillNew + "\\001" + jobDegreeNew + "\\001" + cityNameNew + "\\001" + welfareListNew;

} else {

jobCleanData = jobCleanData + jobIdNew + "\\001" + jobNameNew + "\\001" + salaryNew + "\\001" + jobExperienceNew + "\\001" + skillNew + "\\001" + jobDegreeNew + "\\001" + cityNameNew + "\\001" + welfareListNew + "\\n";

}

}

return jobCleanData;

}

public static String parseJSON(JSONArray skills) throws JSONException {

if (skills == null || skills.length() == 0) {

return null;

}

String skillName = "";

for (int i = 0; i < skills.length(); i++) {

skillName += skills.getString(i) + "-";

}

if (!skillName.isEmpty()) {

return skillName.substring(0, skillName.length() - 1);

} else {

return null;

}

}

public static String salaryResult(String salary) {

if (salary.isEmpty()) {

return "";

}

StringBuffer buffer = new StringBuffer("");

if (salary.contains("K")) {

return salary.substring(0, salary.indexOf("K"));

} else if (salary.contains("元")) {

String s = salary.substring(0, salary.indexOf("元"));

String\[\] split = s.split("-");

for (String temp : split) {

int i = Integer.parseInt(temp);

buffer.append(i \* 30 / 1000 + "-");

}

return buffer.toString().substring(0, buffer.length() - 1);

} else {

return "0-0";

}

}

public static String parseString(String string) {

if (string == null || string.equals("N/A")) {

return "N/A";

}

return string.replace(",", "/").replace("，", "/").replace(" ", "/").replace("-", "/");

}

public static String exField(JSONObject job,String fieldName) throws JSONException {

if (job.has(fieldName) && !job.isNull(fieldName)) {

return job.getString(fieldName);

}

return "N/A";

}

}

package position;

import org.apache.hadoop.conf.Configuration;

import org.apache.hadoop.fs.FSDataInputStream;

import org.apache.hadoop.fs.FileSystem;

import org.apache.hadoop.fs.Path;

import org.apache.hadoop.io.BytesWritable;

import org.apache.hadoop.io.IOUtils;

import org.apache.hadoop.io.Text;

import org.apache.hadoop.mapreduce.InputSplit;

import org.apache.hadoop.mapreduce.RecordReader;

import org.apache.hadoop.mapreduce.TaskAttemptContext;

import org.apache.hadoop.mapreduce.lib.input.FileSplit;

import java.io.IOException;

public class WholeFileRecordReader extends RecordReader&lt;Text, BytesWritable&gt; {

private FileSplit fileSplit;

private Configuration conf;

private boolean processed = false;

private Text key = new Text();

private BytesWritable value = new BytesWritable();

@Override

public void initialize(InputSplit Split, TaskAttemptContext Context) throws IOException, InterruptedException {

this.fileSplit = (FileSplit) Split;

this.conf = Context.getConfiguration();

this.processed = false;

}

@Override

public boolean nextKeyValue() throws IOException, InterruptedException {

if (!processed) {

Path file = fileSplit.getPath();

FileSystem fs = file.getFileSystem(conf);

FSDataInputStream in = fs.open(file);

byte\[\] contents = new byte\[(int) fileSplit.getLength()\];

in.readFully(contents);

key.set(file.getName());

value.set(contents, 0, contents.length);

IOUtils.closeStream(in);

processed = true;

return true;

}

return false;

}

@Override

public Text getCurrentKey() throws IOException, InterruptedException {

return key;

}

@Override

public BytesWritable getCurrentValue() throws IOException, InterruptedException {

return value;

}

@Override

public float getProgress() throws IOException, InterruptedException {

return processed ? 1.0F : 0.0F;

}

@Override

public void close() throws IOException {

}

}

from salary_tmp1;

3）计算平均薪资

create table salary_avg as

select city, round(avg(salary_avg),2) as savg

from salary_tmp2

group by city

order by savg DESC;

4）按薪资范围拆分薪资

由于 NULL 值和 CTAS (Create Table As Select) 推断机制，该语句需要分步创建表，且需要过滤 NULL 值，否则排序键 (sort_key) 会归零，代码如下：

\--预先创建表

create table salary_count_tmp (

sort_key int,

salary_range string

)

row format delimited

fields terminated by '\\001';

\--插入数据

insert into table salary_count_tmp

select

cast(floor(salary_avg / 5) \* 5 as int),

concat(

cast(floor(salary_avg / 5) \* 5 as int),

'-',

cast((floor(salary_avg / 5) + 1) \* 5 as int),

'k'

)

from salary_tmp2

where salary_avg is not null;

5）合并薪资，得到薪资范围表

create table salary_count as

select

sort_key,

salary_range,

count(\*) as num

from salary_count_tmp

group by sort_key, salary_range

order by sort_key asc;

# 数据可视化

## SSM 简介

Java SSM 框架即指 Spring + Spring MVC + MyBatis 的简称，框架集由 Spring、 MyBatis 两个开源框架整合而成（Spring MVC 是 Spring 中的部分内容）,常作为数据源较简单的 web 项目的框架。

各层之间的关系，数据访问层（DAO层）、Service 层这两个层次都可以单独开发，互相的耦合度很低，完全可以独立进行，这样的一种模式在开发大项目的过程中尤其有优势。

Controller，View层因为耦合度比较高，因而要结合在一起开发，但是也可以看作一个整体独立于前两个层进行开发。这样，在层与层之间我们只需要知道接口的定义，调用接口即可完成所需要的逻辑功能，一切显得非常清晰简单。

Service 层是建立在 DAO 层之上的，建立了 DAO 层后才可以建立 Service 层，而 Service 层又是在 Controller 层之下的，因而 Service 层应该既调用 DAO 层的接口，又要提供接口给 Controller 层的类来进行调用，它刚好处于一个中间层的位置。每个模型都有一个 Service 接口，每个接口分别封装各自的业务处理方法。

相比于之前的 SSH（Spring + Struts + Hibernate），SSM 更加轻量化和灵活，是目前业界主流的 Java Web 开发框架。为了方便理解，如图所示：
