# jobSSM 项目问题修复总结

## 项目架构

Spring + Spring MVC + MyBatis 三层架构，数据库 MySQL (`job` 库)，Tomcat 9 部署。

**Context 分层：**
- **Root WebApplicationContext** (`ac.xml`)：管理 Service、Mapper、数据源
- **Servlet WebApplicationContext** (`springMVC.xml`)：管理 Controller、MVC 配置

---

## 修复问题 1：包名配置错误导致 Bean 找不到

**现象：** 启动报 `NoSuchBeanDefinitionException: No qualifying bean of type 'org.service.CityService'`

**根因：** `ac.xml` 中的 component-scan 和 MapperScanner 包名写成了 `org.example.service` 和 `org.example.mapper`，实际代码在 `org.service` 和 `org.mapper`。

**修复：** `src/main/resources/ac.xml`
```xml
<!-- 改前 -->
<context:component-scan base-package="org.example.service"/>
<property name="basePackage" value="org.example.mapper"/>

<!-- 改后 -->
<context:component-scan base-package="org.service"/>
<property name="basePackage" value="org.mapper"/>
```

---

## 修复问题 2：Bean 的 package 声明错误

**现象：** 部分 Bean 类编译后在其他包中找不到

**根因：** 5 个 Bean 文件写的是 `package bean;`，正确应为 `package org.bean;`

**涉及文件：**
- `SalaryAvgBean.java`
- `SalaryBean.java`
- `JobNameBean.java`
- `SkillCountBean.java`
- `WelfareCountBean.java`

---

## 修复问题 3：数据库表名与 Mapper SQL 不一致

**根因：** 参照项目 `jobDataView` 中 DAO 使用的表名与实际 MySQL 数据库 (`job` 库) 中的表名有差异。以下通过 SQL dump 确认了真实表结构。

| 功能 | 代码中表名（错误） | 实际表名（正确） |
|------|-------------------|-----------------|
| 工作经验分布 | `experience_count` | **`exp_count`** |
| 技能需求统计 | `skills_count` | **`skill_count`** |
| 岗位名称统计 | `jobName_count` | **`jobname_count`** |

**涉及 Mapper 文件：** `ExpMapper.java`、`SkillMapper.java`、`JobNameMapper.java`

---

## 修复问题 4：列名映射与排序字段

**薪资区间分布 (`/salary`)：**
- `sort_key` 列不存在 → 改为使用 `min_salary` 列排序：`order by min_salary ASC`

**城市平均薪资 (`/avgSalary`)：**
- DAO 中误导性地使用了 `rs.getDouble("savg")`，但实际 DB 列名就是 `salary_avg_city`
- 确认 `select * from salary_avg` 即可正确映射到 `SalaryAvgBean`

**Service 层 null 保护：**
- `SalaryAvgService` 增加了防御性 null 检查，防止 MyBatis 返回异常数据时 NPE

## 8 个数据接口总览

| URL | 图表类型 | Mapper | Service |
|-----|---------|--------|---------|
| `/city` | 饼图 | `CityMapper` | `CityService` |
| `/degree` | 饼图 | `DegreeMapper` | `DegreeService` |
| `/experience` | 饼图 | `ExpMapper` | `ExpService` |
| `/avgSalary` | 柱状图 | `SalaryAvgMapper` | `SalaryAvgService` |
| `/salary` | 柱状图 | `SalaryMapper` | `SalaryService` |
| `/jobName` | 词云 | `JobNameMapper` | `JobNameService` |
| `/skills` | 词云 | `SkillMapper` | `SkillService` |
| `/welfare` | 词云 | `WelfareMapper` | `WelfareService` |

**返回格式：**
- 饼图/词云接口返回 `DataBean[]` → `[{"name":"xxx","value":"yyy"},...]`
- 柱状图接口返回 `Object[]` → `[["标签1","标签2"],[值1,值2]]`

## 前端注意事项

- **ECharts 重复初始化：** 切换图表时需先 `echarts.dispose(dom)` 再 `echarts.init(dom)`，否则报 "instance already initialized"
- **jQuery AJAX：** Spring `@ResponseBody` 返回 `application/json`，jQuery 会自动反序列化，**不要**在 success 回调中再调用 `JSON.parse()`
- **JS 资源路径：** 通过 `<mvc:resources mapping="/js/**" location="/js/"/>` 映射，JSP 中用 `${pageContext.request.contextPath}/js/xxx` 引用
